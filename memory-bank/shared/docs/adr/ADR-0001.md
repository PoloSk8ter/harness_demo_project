# ADR-0001 — Build on adk_studio's canonical orchestrator + specialists shape

- **Status:** accepted
- **Date:** 2026-06-25
- **Architecture version:** v1

## Context
The brief requires a guardrail → route → one-specialist → finalize composition over four banking
competencies, on simulated data. The company framework library (`framework_library.path`) admits exactly
one relevant framework: **`adk_studio`**, the team's standard layer over Google ADK 1.16, whose *canonical*
multi-agent shape is `make_sequential([triage, router, synthesizer])` with specialists attached to the
router. The brief's own headline case-against asks whether a single tool-using agent would match a
multi-specialist system at lower latency and cost.

## Decision
Implement the assistant as the adk_studio canonical shape: a `SequentialAgent` of a guardrail/triage agent,
a router agent (holding the specialists + the simulated-store tools), and a synthesizer agent — every agent
constructed through `make_agent` / `make_sequential` (never a raw `LlmAgent`). The allowlist check passes
(`check_allowlist(['adk_studio']) == []`); no `vet-lib` is needed (adk_studio was ADMITTED 2026-06-24).

## Alternatives rejected
- **A single capable agent with tools + a safety prompt** — the brief's headline falsifier. Likely cheaper
  and lower-latency, but the *deliverable here is the multi-specialist composition itself*, and the
  company's blessed shape is orchestrator + specialists. Rejected for this build, but kept live as the
  baseline the acceptance script (ADR-0004) can later measure routing accuracy/latency against — the
  falsifier is respected, not dismissed.
- **Raw `google-adk` `LlmAgent`** — violates the adk_studio convention "construct every agent through the
  factory — never raw `LlmAgent`," and loses telemetry, fail-safe callbacks, and structured-output safety.
- **A pure-Python stdlib reimplementation** — deterministic, but reinvents the commodity orchestration shell
  and never exercises the company agent stack this reference exists to demonstrate.

## Consequences
Pins `google-adk==1.16.0` + Gemini (per-tier from env). The Junior follows the adk_studio
`conventions.md` + `exemplars/` (orchestrator + leaf-agent) and the router/synthesizer split. A
change-request against this ADR would have to overturn the framework choice itself (e.g. evidence the
single-agent baseline wins on the acceptance script), and would re-circulate the architecture to the SA.
