# ADR-0004 — Two-tier verification: deterministic seam tests + tolerant real-model evals

- **Status:** accepted
- **Date:** 2026-06-25
- **Architecture version:** v1

## Context
The brief's acceptance is a 6-check scripted conversation that must be observable and pass/fail. But
adk_studio's eval harness (`run_agent`) drives the **real** model (Gemini): exact wording is
non-deterministic, runs spend tokens, and the framework's own conventions say *do not* wire it as a
token-spending CI gate until an offline/mock path exists (tracked backlog item F2). Meanwhile the safety
mechanics (ADR-0002) are pure code and *can* be tested deterministically. Verification must reconcile the
two without making the safety guarantees depend on a flaky model run.

## Decision
Split verification into two tiers:
- **Deterministic seam tests (no model) — the default suite and the CI gate.** `pytest -q -m "not llm"`
  covers: simulated-store correctness; `set_card_state` confirmation-token enforcement; schema validation
  (`EscalationRecord`, `FinalReply`, `RouteDecision`, `CardActionDraft`); and the guardrail callback's
  fail-closed behaviour. These cover the ADR-0002 safety mechanics and are 100% deterministic.
- **Real-model evals (`run_agent`) — opt-in, observable-signal assertions.** Cover routing accuracy,
  refusal behaviour, and the end-to-end acceptance script; assert on `.tool_calls()`, `.state` flags,
  schema-field presence, and no-state-change — **never** exact strings. Marked `@pytest.mark.llm`, excluded
  from the default suite, run deliberately/locally.

`HARNESS-CONTEXT.stack.test_command` is therefore `pytest -q -m "not llm"`, and the `@pytest.mark.llm`
marker convention is locked (Invariant 9).

## Alternatives rejected
- **String-match the model output to make the whole acceptance deterministic** — brittle; fights
  non-determinism; produces false failures.
- **Wire `run_agent` as the CI gate** — spends tokens, flaky, and contradicts adk_studio's own caveat.
- **Skip model evals entirely** — the brief's acceptance is end-to-end behavioural; deterministic tests
  alone don't prove routing or refusal.

## Consequences
The safety guarantees are provable deterministically and gate CI; the NLU is *measured* (not vibed) but
kept off the blocking path. If adk_studio later adds an offline/mock model path (its F2 backlog), the
acceptance suite could join CI — that change would arrive as a change-request to this ADR.
